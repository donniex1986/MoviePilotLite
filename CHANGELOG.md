# Changelog

## 2026-04-01
- 下载器配置列表样式重构：卡片信息层级与运行指标区重新设计，增强与站点管理页的视觉区分。
- 下载器指标区新增合并分栏展示（总下载、总上传、上/下比），并统一“上/下比”计算口径为累计上传/累计下载。
- 下载器配置列表新增隐私模式开关，支持对 URL 地址信息进行模糊遮挡显示。
- 隐私模式开关调整为 `title + switch` 交互样式，并支持本地持久化恢复。
- 媒体整理页加载链路优化：进入页面优先展示本地缓存，再请求远程最新数据，减少首屏等待体感。
- 媒体整理页新增历史缓存读写能力（按关键词维度缓存），远程请求失败时保留已展示数据避免列表闪空。
- 媒体整理页下拉刷新改为并行刷新存储与历史数据，缩短刷新耗时。
- 媒体整理页信息栏加载态调整为左侧 `CupertinoActivityIndicator`，移除右上角独立 loading 图标。
- 媒体整理记录项操作模式文案本地化：`move/copy/hard link/soft link` 显示为中文。

## 2026-03-31
- iOS 影视推荐 Widget 请求解析增强，兼容多种返回结构（`data/results/items/list/subjects/rows`），修复部分场景下空白显示问题。
- iOS 影视推荐 Widget 点击行为调整为条目级跳转，点击不同卡片可直达对应媒体详情页，不再固定进入同一详情。
- iOS Widget 深链路由补充 `media-detail` 解析，支持从 Widget 冷启动进入 App 后直接打开媒体详情页。
- 登录与自动登录流程补充 Widget 主动刷新时机，降低推荐 Widget 停留“请先登录”状态的概率。
- 新增 Widget 关键链路日志（会话读取、请求状态、解码结果、路由解析），便于排查推荐数据加载异常。
- iOS Widget 代码按功能拆分：新增独立 `WidgetAppIntents.swift`，集中管理 App Intent 与 Timeline Provider。
- 应用设置页新增“更新日志”入口，支持直接查看应用 Changelog。
- Changelog 页面改为 Markdown 渲染，保留标题与列表等格式化展示。
- Changelog 默认仅展开最新版本更新，历史版本改为折叠展开样式以提升可读性。
- 新增“下载器直连下载”开关（默认关闭），下载弹层支持一键直连下载器提交本地种子文件。
- 直连下载开关增加启用提醒：由于站点特殊性不能保证适配所有站点，如有特殊需求请提交 PR。
- 下载弹层底部操作区重构：新增直连下载按钮、主下载按钮与单按钮铺满布局，并优化键盘遮挡与状态残留问题。
- 直连下载页支持查看本地种子文件路径列表，便于排查本地文件来源与提交路径。
- qBittorrent 日志页改造为底部 Floating Bar 交互，整合搜索、筛选、排序入口并补充底部防遮挡留白。
- qBittorrent 日志筛选弹层改为开关即时生效，移除取消/应用二次确认流程。
